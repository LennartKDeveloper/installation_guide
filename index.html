Flutter Setup Guide

Wähle dein Betriebssystem für den Schnelleinstieg

Windows Environment

Der direkte Weg zur Flutter-Entwicklung auf Windows.

1. Vorbereitung

  • Installiere Git for Windows. Wähle im Setup "Use Git from the Windows Command Prompt".
  • Lade Visual Studio Code herunter und installiere es.

2. Flutter SDK Einrichtung

  1. Lade das Flutter SDK (Stable Zip) von der offiziellen Website.
  2. Entpacke es z.B. nach C:\src\flutter.
  3. Path Variable: Suche in Windows nach "Umgebungsvariablen", bearbeite "Path" und füge C:\src\flutter\bin hinzu.

3. VS Code Extensions

  • Öffne VS Code, gehe zum Extension-Marktplatz.
  • Suche und installiere die Extension Flutter (Dart kommt automatisch mit).

4. Verifizierung

Öffne ein Terminal und führe aus:

flutter doctor

macOS Environment

Optimiere deinen Mac für die App-Entwicklung mit Flutter.

1. Vorbereitung

  • Installiere Xcode aus dem App Store (wichtig für iOS-Simulator).
  • Öffne Xcode einmal, um die Lizenzbedingungen zu bestätigen.
  • Installiere Visual Studio Code.

2. Flutter SDK Einrichtung

  1. Lade das Flutter SDK für macOS herunter.
  2. Entpacke es in deinen Home-Ordner, z.B. ~/development/flutter.
  3. Path Update: Füge dies zu deiner .zshrc oder .bash_profile hinzu:
export PATH="$PATH:$HOME/development/flutter/bin"

3. Extensions & CocoaPods

  • Installiere die Flutter Extension in VS Code.
  • Installiere CocoaPods für iOS-Abhängigkeiten via Terminal:
    sudo gem install cocoapods

4. Verifizierung

Prüfe deine Installation im Terminal mit:

flutter doctor

GitHub & VS Code Integration

Versionsverwaltung ist kein Luxus, sondern Pflicht. So geht's.

1. Git Konfiguration

Öffne das Terminal in VS Code und setze deine Identität:

git config --global user.name "Dein Name" git config --global user.email "mail@example.com"

2. Account verbinden

  • Klicke in VS Code unten links auf das Profil-Icon.
  • Wähle "Sign in to Sync Settings" (oder GitHub) und autorisiere den Browser-Zugriff.

3. Der Workflow

  1. Öffne dein fertiges Flutter-Projekt.
  2. Gehe zum Source Control Tab (links).
  3. Klicke auf "Publish to GitHub". VS Code erstellt das Repo, macht den ersten Commit und lädt alles hoch.
